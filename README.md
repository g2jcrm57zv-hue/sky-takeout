项目简介
苍穹外卖是一个为餐饮企业定制的软件产品，包括系统管理后台和移动端应用两部分。其中系统管理后台主要提供给餐饮企业内部员工使用，可以对餐厅的分类、菜品、套餐、订单、员工等进行管理维护，并提供多维度的数据统计与报表导出功能；移动端应用主要提供给消费者使用，可以在线浏览菜品、添加购物车、下单等。


技术选型

本项目采用前后端分离的开发模式。

后端技术栈：

核心框架： Spring Boot

持久层框架： MyBatis / MyBatis-Plus

数据库： MySQL

缓存： Redis

报表导出： Apache POI

其他工具： JWT (身份认证), Swagger/Knife4j (接口文档), Lombok

前端技术栈 (参考)：

管理端：Vue.js + ElementUI

用户端：微信小程序原生开发

---

核心功能模块

1.  数据统计与报表模块 (核心亮点)

提供详尽的营业数据分析支持，辅助商家进行运营决策。

营业额统计： 支持自定义时间区间，折线图展示每日营业额变动。

用户统计： 统计每日新增用户数与总用户数，掌握用户增长趋势。

订单统计： 多维度展示有效订单量与订单完成率。

销量排名： 统计指定时间内的菜品/套餐销量 Top 10。

Excel 报表导出： 结合 Apache POI 技术，支持将近30天的运营数据一键导出为格式化的 Excel 报表，方便财务与运营人员存档与线下分析。

2.  员工与权限管理

员工的登录与退出。

员工信息的增删改查及账号状态的分页展示与启用/禁用。

3.  菜品与套餐管理

分类管理： 菜品分类与套餐分类的维护。

菜品管理： 菜品及其口味的维护，支持图片上传，起售/停售状态切换。

套餐管理： 包含多个菜品的套餐组合维护。

4. 订单管理

用户端： 浏览菜单、加入购物车、微信支付下单、历史订单查询、取消订单。

管理端： 订单接单、拒单、派送、完成以及订单状态的实时监控。

环境搭建与运行
1. 环境要求
JDK 1.8+
MySQL 5.7+
Redis 5.0+
Maven 3.6+
2. 数据库准备
登录 MySQL，创建数据库 sky_take_out。
执行项目 sql 目录下提供的初始化脚本 sky.sql。
3. 启动步骤
克隆项目：
Bash

git clone [https://github.com/g2jcrm57zv-hue/sky-takeout]
配置修改：

修改 sky-server/src/main/resources/application-dev.yml 文件中的配置信息：

MySQL 的数据库连接地址、用户名和密码。

Redis 的连接地址与密码。

小程序相关的 appid 与 secret（如果需要测试小程序端登录和支付）。

启动 Redis 服务。

运行项目：

找到 sky-server 模块下的启动类 SkyApplication，运行 main 方法。

访问接口文档：

启动成功后，浏览器访问 http://localhost:8080/doc.html 查看后端接口文档。
